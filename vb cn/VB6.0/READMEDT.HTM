Microsoft Visual 数据库工具中的新特征

要了解这一版本的 Microsoft Visual 数据库工具中的新特征，请在 MSDN 库 Visual Studio 6.0 文档的目录中查找“Visual 数据库工具的新特征”，这部分位于 Visual Studio 文档节点的参考部分。

已经发现的问题

在现在这个版本的 Visual 数据库工具中已经发现了下面这些问题。对于与特定产品相关的问题，请参阅相应产品的自述文件。

只支持最新的 MS ODBC 和 OLE DB 驱动程序

即使旧版本的驱动程序和第三方驱动程序可以和 Visual 数据库工具一起使用， Visual 数据库工具只保证能够支持 Microsoft ODBC 和 OLE DB 驱动程序。如果您遇到任何意外的情况，请查证所使用的驱动程序是正确的。

当连接到一个国别设定的数据库时显示不正确的数据

当连接到一个国别设定的数据库时显示不正确或者无效的字符，请检查计算机的区域设置以确保其与该数据库中的数据匹配。

需要 SQL Server Service Pack 3

要对 SQL Server 6.5 使用 Visual 数据库工具需要 SQL Server Service Pack 3。在使用 Visual 数据库工具之前，请确保服务器上已经安装了 Service Pack 3 或者更高版本。

下面的问题与 Microsoft(R) Visual Basic(R) 有关

对 Oracle 数据库运行生成表查询

如果使用 Microsoft OLE DB provider for Oracle，对一个 Oracle 数据库的生成表查询就不能正确地运行。请改用 Microsoft ODBC 驱动程序来创建与这种数据库的连接。

默认情况下并不显示所有的查询窗格

当使用数据视图打开一个表时，实际上是用一条“SELECT * FROM table”命令打开查询设计器。要在 Visual Basic 中打开“图表”、“网格”和 SQL 窗格，请从视图菜单中单击显示窗格。这样就可以选择打开或者关闭哪些窗格。
