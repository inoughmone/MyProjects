Overview

PowerTransfer is an extension to PowerDesigner to allow the transfer of data into Sybase Adaptive Server Enterprise.  The source database can be any ODBC datasource supported by PowerDesigner.  The program uses the Sybase bulk copy mechanism for high performance inserts into Sybase ASE.  

The program has been tested with the following data sources: Sybase ASE, Oracle 9, Microsoft SQL Server 2000, Microsoft Access

System Requirements

In order to install and use PowerTransfer, your system will require:

  • Sybase PowerDesigner 9.5.2
  • Sybase Open Client 12.5

Installation
  • Unzip the distribution file into a temporary directory.
  • If your system does not have the Microsoft .NET framework installed already, install it by clicking dotnetfx.exe.
  • Click Setup.exe to install PowerTransfer.
  • PowerTransfer should be accessible through the PowerDesigner Tools->Execute Commands menu.

Steps For Copying a Database

  1. Before starting, you will need to set up the following:
  • The database in the target ASE should be created and configured to allow bulk operations.
sp_dboption mydatabase,"select into", true
  • Configure an ODBC connection to your source database for use by PowerDesigner and PowerTransfer.
  • Configure an ODBC connection to the target Sybase ASE for use by PowerDesigner.
  • Sybase Open Client should be configured for a connection to the target ASE.  Use the Sybase Dsedit utility to do this.  This connection will be used by PowerTransfer to perform bulk inserts of the data.

  1. Using PowerDesigner, you will need to create the database on the target ASE:
  • Create a physical model of the source database by using the Reverse Engineer feature of PowerDesigner.
  • Change the Current Database Type of the physical model to Sybase ASE 12.5.
  • Examine the converted schema and make any changes required.
  • Connect to the target ASE and generate the database.  
NOTE: The database should first be created without indexes and rules. These items should be added after the data has been moved.
  1. Start PowerTransfer from the Tools->Execute Commands menu of PowerDesigner.
  2. Connect to the Source and Target database from the Server Login tab.
  3. In the Table Copy tab, select the tables you wish to transfer and click the Add button to queue up the transfer.
  4. The number of rows in the source table should be listed for each source table.
  5. Click Clear List if you change your mind regarding the tables to be copied.
  6. Click Start Copy to initiate the copy

Server Login Panel

This panel describes source and target connection settings.  The source connection is an ODBC DSN value.  The target connection is a Sybase Open Client connection.  Use the Sybase Dsedit utility to configure these connections.  By default, one connection will be opened to the source and destination.  Change the Number of Connections property in the Options tab to enable multiple simultaneous transfers through more than one connection.

Table Copy Panel

The list on the left shows the table definitions that are defined in the currently open physical model.  The name of the physical model is displayed at the top of the table.  Click the Load Table List From Active Model button if you change models.  By default, only tables are displayed, however views may also be displayed if this option is selected in the Options tab.  The tables are listed in an order determined by their primary and foreign key relationships so that tables are listed after their dependencies.  It is recommened however that for performance reasons, transfers be performed with no referential integrity applied in the target.

When Start Copy is clicked tables will be transferred in the order they are listed.  If multiple connections have been configured, multiple simultaneous transfers will commence and execute in a non-deterministic order.


Options Panel

Changes to ASE bulk insert and connection properties do not take affect until connections are closed and reopened.

Rows Per Bulk Insert
This determines the number of rows fetched from the source and pushed to the target in a single bulk operation.  Increasing this value uses more memory but increases performance.

Rows Per Commit
This number determines the frequency that a batch commit is sent to an ASE.

Number Of Connections
This number determines the number of connections to be opened to the source and destination servers.  More than one connection enables simultaneous transfers.

Packet Size
This option determines the Sybase TDS packet sized used to connect to the target ASE.  Larger packet sizes provide better performance, however use more server memory.  This option should not be set higher than the maximum configured packet size configured in your target server.  The maximum allowed packet size on ASE can be configured using:
sp_configure "max network packet size",8192

Truncate Target Table
This checkbox indicates whether a TRUNCATE TABLE statement should be sent to the target at the start of  the transfer.

Show Views In Table List
This checkbox determines whether views in the physical model should be listed as a source tables.  This option allows a view in the source database to be copied to a table in the target.  This can be useful for moving a subset or an aggregation of the source database's data.


Schema Conversion Notes

General
  • Column names may be changed between the source and the target, however the column ordering must be the same.
  • Table names cannot altered and must be valid names on ASE.  
  • The table owner is assumed to be as described in the PowerDesigner physical model.  If you want to have different table owners between the source and target, you can assign the owner in the PowerDesigner physical model to NONE and let the userid default to the id that is used for the ODBC/Open Client connections.
  • Data types can be different between the source and the target as long as the types are convertable.  You may receive overflow errors during a transfer if copying into a smaller sized type.
  • Use views in the source database if you wish to copy subsets of the source tables or if you wish to alter the table names.  The target database should contain an actual table definition that matches the column list of the view.
Microsoft SQL Server
  • Watch for the identity property being applied to non-numeric datatypes.  ASE requires a NUMERIC type for the identity property.
  • Uncheck "Physical options" in the PowerDesigner database generation panel to avoid "ON PRIMARY" being generated on the CREATE statements generated for ASE.
Oracle
  • XML and TIMESTAMP datatypes are not supported.
