If the Text Source property is "Local File", setting this property to "True" causes the panel to include any localized versions of File Name in the archive for use during runtime.

If set to "True", then ensure that the correctly named localized files are located in the same directory as the default file. A localized file must comply with the class naming convention pattern described in Sun's API documentation for Resource Bundles (http://java.sun.com/j2se/1.4.1/docs/api/java/util/ResourceBundle.html), e.g., "myfile_en.txt" (for English), "myfile_es.txt" (for Spanish), etc.

NOTE: The list of localized files to be included in the archive is determined by the languages specified under Language Support in the Build Tab. For a list of supported languages and their associated Java abbreviation, see the MultiPlatform User's Guide, Chapter 4.0 Build Tab/Language Support.
