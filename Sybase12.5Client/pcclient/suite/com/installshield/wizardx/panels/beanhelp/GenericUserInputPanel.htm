The User Input Panel is used to retrieve user input such as name, license key number, email address, URL, etc. It can also be used to validate that the end user is authorized to install the product(s), and/or to obtain the information required to notify them of future patches, upgrades, etc. This bean is extremely flexible and can be customized to obtain any type of information from the end user for use/storage later during the installation/uninstallation.

For example, to validate that the end user is authorized to install the product, add this panel after the Welcome Panel and request authorization information. Then add a User Input Condition to the Install Action bean that requires that it "Must be met". The user input is stored as an extended property with the key being the name of the input field, i.e., input type, that can be retrieved at any point after user input during the installation. The developer sets the name for each input field, each representing a different piece of information that the end user entered, then retrieves the value of each input field using the "$W" string resolution method.

For example:

$W(beanID.inputFieldName)

Where:

beanID is the bean ID of the User Input Panel, and inputFieldName is the value of the Name property, specified by the developer for that input field on the panel.
For additional information, see the MultiPlatform User's Guide, Chapter 3.4 - Wizard Panels/User Input/Validating User Input.
